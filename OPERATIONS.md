# JUNICHI v1.0 運用メモ

## 対象

英検準一級合格アプリ準一 (JUNICHI) の Phase 1 MVP 運用メモです。

---

## 現状仕様

- 公開方式: GitHub Pages
- 実装形式: PWA
- 主機能: 語彙オーディオクイズ
- 進捗保存: `localStorage`
- build識別: 画面下の `build:` 表示

---

## 推奨利用環境

### 推奨ブラウザ

- **Google Chrome (Android)**
  - 実機で音声再生を確認済み

### 非推奨ブラウザ

- **DuckDuckGo Browser (Android)**
  - 最新 build は表示される
  - ただし Web Speech API の制限により、音声が再生されない場合がある

---

## 不具合切り分け手順

### A. 画面が古い

確認ポイント:

- `build:` 表示が想定どおりか
- シークレットタブではどうか
- 別ブラウザではどうか

対応:

- ブラウザキャッシュ削除
- PWA をホーム画面追加している場合は削除して再追加
- GitHub Actions の最新 deploy 成功確認

### B. 音声が鳴らない

確認ポイント:

1. `build:` が最新か
2. Chrome では鳴るか
3. DuckDuckGo など別ブラウザではどうか
4. 1回目タップ後、2回目で鳴るか

判断:

- Chrome で鳴る → アプリは正常、ブラウザ差異の可能性が高い
- どのブラウザでも鳴らない → 実装または端末権限制約を再調査

### C. ボタンが押せない / 無反応

確認ポイント:

- 旧キャッシュを見ていないか
- build 表示が更新されているか
- PWA 化されたホーム画面版を使っていないか

---

## デプロイ確認手順

1. GitHub で `main` にコミット
2. Actions で最新 workflow の `build` と `deploy` が両方 Success か確認
3. 公開URLを開く
4. 画面下の `build:` 表示を確認
5. Chrome 実機で最低限の動作確認

---

## 最低限の受け入れ確認

- 画面が開く
- 問題が表示される
- 4択が押せる
- 正誤進行できる
- 進捗が保存される
- Chrome で音声が鳴る
- build 表示が最新

---

## 既知の制限

- ブラウザ差異により音声再生可否が変わる
- DuckDuckGo Browser では音声再生制限が出る可能性あり
- Phase 1 は語彙クイズ中心で、他技能は未実装
- PWA キャッシュの影響で旧版が残る場合がある

---

## 運用方針

- **音声学習は Chrome を基準ブラウザとする**
- 不具合報告時は、必ず `build:` 番号とブラウザ名をセットで記録する
- 端末差より先に、まずブラウザ差とキャッシュを疑う
- 仕様境界である可能性と実装不具合を分けて判断する

---

## 次フェーズへ向けた課題

- 音声対応ブラウザの明示
- 画面上で「推奨: Chrome」を表示するか検討
- DuckDuckGo など非推奨ブラウザ時の案内文改善
- 将来的には Capacitor 化してブラウザ差を減らす
